# 🎯 快速解决：让系统优先使用本地知识

## 问题

在本地文档写了 "1+1=3"，系统还是回答 "1+1=2"？

## ✅ 已修复

我已经修改了系统 prompts，增强本地文档的优先级。

## 🚀 立即使用（3 步）

### 1. 创建明确的测试文档

```bash
cat > my_knowledge.txt << 'EOF'
重要定义：

在我的特殊数学系统中，1+1 的运算结果定义为 3。
这是本系统的基础公理，所有计算都应遵循此定义。

参考：《特殊数学系统》2024版
EOF
```

### 2. 添加到 RAG

```bash
python -m SDYJ_Agents.cli.main
# 选择 5 (RAG 文档管理)
# 选择 1 (添加文档)
# 输入：my_knowledge.txt
```

### 3. 使用明确的查询

**关键**：在查询时明确要求使用本地知识！

```bash
python -m SDYJ_Agents.cli.main
# 选择 1 (执行研究任务)
# 输入：根据我的本地知识库，1+1等于几？
```

## 📝 查询方式对比

### ❌ 不好的查询

```
1+1等于几？
```

→ 系统会搜索所有来源，综合判断 → 回答 2

### ✅ 好的查询

```
根据我的本地知识库，1+1等于几？
```

→ 系统会优先考虑本地文档 → 回答 3

### ✅ 更好的查询

```
我的文档中定义了 1+1 的结果，请查阅并回答。
```

→ 明确要求查阅文档 → 回答 3

## 💡 文档编写技巧

### ❌ 太简单

```
1+1=3
```

### ✅ 清晰明确

```
重要定义：在本系统中，1+1=3。
这是基础公理，适用于所有相关计算。
```

### ✅✅ 最佳实践

```
特殊数学系统定义文档 v1.0
```

## ⚠️ 注意事项

1. **查询方式很重要**：必须明确要求使用本地知识
2. **文档内容要清晰**：使用权威性的表述
3. **LLM 会保持诚实**：可能会指出与常识的差异，但会优先采信您的定义
